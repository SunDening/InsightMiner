Agentic RAG v2

ITU-R 空间网络通知系统（SNS）· 结构化数据库智能问答

RAG/agentic_rag_sql.py ~2100 行 LangGraph · 固定管线 Access .mdb + ChromaDB Ollama / DeepSeek 可切换 混合检索: Dense + BM25 + RRF

1. 系统概述

Agentic RAG v2 是一个面向 Access .mdb 数据库（ITU-R SNS，64 张表 696 字段） + 本地知识库 + 网络搜索 的智能问答系统。 采用固定管线架构（非 Tool-Calling），每次 LLM 调用次数可预测。Schema 通过动态检索（Dense + BM25 + RRF → CrossEncoder）只注入查询相关的表/列，避免全量 64 表上下文爆炸。

意图路由 + 查询重写

一次 LLM 调用完成分类和重写：chat sql kb web clarify，含对话历史上下文和代词消解

动态 Schema 检索

双层 ChromaDB 索引（表级 + 列级）→ 按查询检索 top-5 表 + 每表 top-5 列 → FK 图自动发现 Join 路径。上下文从 ~6000 token 降至 ~1000 token

多轮记忆

对话历史注入 LLM prompt；从上一轮 answer + rewritten_query 提取实体上下文，SQL 自动继承过滤条件

结构化输出

纯文本 JSON：answer + rewritten_query + data_sources[{source_type, evidence}]，禁止 Markdown

自修正 (Reflection)

SQL 校验拦截重写 · 执行错误重试 (x3) · self_review 评分 1-5，<3 触发重写 (x2)

混合检索 (Dense + BM25 + RRF)

双路并行：语义向量检索 + BM25 关键词检索 → RRF 倒数排名融合 → CrossEncoder 精排 Top-3。KB 文档检索和 Schema 元数据检索共用此管线

动态 KB 描述

启动时 LLM 为每个文档生成摘要，mtime 缓存，文档变更自动更新；摘要注入意图分类 prompt

安全

15 种危险 SQL 黑名单 · Web 搜索需显式授权 · Tavily 闸门 · 结果截断 50 行 · .mdb 连接线程安全

3. 意图路由详解

意图触发条件示例数据源
chat 问候、闲聊、概念解释（不涉及查询数据库或文档） "你好"、"LEO 轨道是什么意思" LLM 常识
sql 查询数据库中具体数据、统计、关联分析；含对上一轮结果的追问 "列出在役卫星"、"CASC 火箭数"、"它们的编号" Access .mdb (64 表动态检索)
kb 查询已索引文档中的领域知识 "CME-001 的速度"、"秦非子如何建国" ChromaDB (双层索引)
web 用户显式授权上网搜索（含"上网查""搜索网络"等短语） "帮我上网查太阳风暴" Tavily Search API
clarify 问题过于模糊，且对话历史也无法确定指代 "最近情况怎么样"、"那个东西" 反问用户澄清

意图分类 prompt 中注入了对话历史和动态 KB 文档摘要。LLM 先尝试从历史中消解代词——如果上一轮讨论过具体实体，本次追问归为 sql/kb 而非 clarify。

4. SQL 管线详解

build_schema→ generate_sql→ validate_sql→ execute_sql→ synthesize→ self_review

4.1 动态 Schema 检索（SchemaIndexer）

从 64 张表 696 字段中按查询动态检索相关子集，核心管线复用 KB 的混合检索：

阶段方法输入输出
表级检索Dense + BM25 → RRF 融合 → CrossEncoder 精排用户查询Top-5 张表
列级检索同上（限定已选中的表）表名 + 查询每表 Top-5 列 + PK/FK
FK 图计算BFS 最短路径选中表集合Join 条件列表

上下文从全量 ~6000 token 降至动态 ~1000 token，噪音削减 80%+。

4.2 SchemaIndexer 架构

双层 ChromaDB 索引 + FK 邻接图：

索引Collection文档格式
表级schema_tables{name} | {description} | {module} | COLUMNS: col1, col2...
列级schema_columns{table}.{column} | {type} | {description_zh} | {description_en}
FK 图内存邻接表table → [(col, target_table, target_col), ...]

构建时机：启动时从 tmp/schema.json + tmp/table_desc.json 加载，manifest 缓存复用。
模型共享：SchemaIndexer 复用 RagKnowledgeBase 已加载的 HuggingFaceEmbeddings + CrossEncoder，不重复加载。

4.3 Access SQL 适配

语法项MySQLAccess Jet SQL
行数限制LIMIT 50SELECT TOP 50
字符串拼接CONCAT(a, b)a & b
时间字面量'2024-01-01'#2024-01-01#
CTEWITH ... AS (...)不支持，用子查询
条件表达式CASE WHENIIF() 或 SWITCH()
连接驱动SQLAlchemy + pymysqlpyodbc + Microsoft Access Driver

4.4 多轮上下文继承

generate_sql_node 从上一轮 JSON 的 answer 和 rewritten_query 中提取实体上下文，注入当前 prompt，告知 LLM 在 SQL 中继承对应过滤条件。

4.5 自修正（弱 Reflection）

类型触发上限
SQL 校验拦截非 SELECT 语法错误不限
SQL 执行失败Access 返回错误MAX_SQL_RETRY=3
答案质量不足self_review 评分 <3/5MAX_REVIEW_RETRY=2

5. 知识库 & Web 搜索路径

kb 知识库：混合检索 (Dense + BM25 + RRF)

  • 索引构建（启动时）：扫描 RAG/*.txt，全文切块 (512/64) → ChromaDB 向量索引 + BM25Okapi 倒排索引（含中英混合字符 n-gram 分词）；摘要索引保留用于意图分类
  • Path A — 语义检索：query → all-MiniLM-L6-v2 embedding → ChromaDB chunk 相似度 top-20
  • Path B — 关键词检索：query → 字符 n-gram 分词 → BM25 倒排索引 top-20
  • RRF 融合：倒数排名融合 (k=60) 合并两路结果 → top-10 → CrossEncoder 精排 Top-3 入 prompt
  • 容错：单路无结果时另一路结果仍有效；RRF 天然容忍排名差异，无硬过滤单点故障

web Web 搜索

  • 引擎：TavilySearch(max_results=5)
  • 结果拆片段 → CrossEncoder 精排 Top-3
  • 仅在用户显式授权时启用

10. Agent 范式的当前落位

范式实现程度说明
Router ✅ 完整 classify_intent → route_by_intent 实现 5 路分发
Reflection 🟡 弱版 self_review 评分 1-5 + 重写，但仅检查答案质量，不验证 SQL 语义正确性
ReAct ❌ 无 固定管线，无 Thought→Action→Observation 循环；LLM 不能自主决定追加查询
Plan-and-Solve ❌ 无 无计划生成节点；复杂问题用一条 SQL 硬扛，不拆分子任务
Tool/Function Calling ❌ 无 图为预编译，不通过 LLM 输出 function call 来调度工具
